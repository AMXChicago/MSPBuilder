# User Flows

## Primary MVP Flow
1. User creates or joins an organization.
2. User completes the founder profile.
3. User defines the target business model.
4. User creates service definitions and assembles service packages.
5. User configures pricing models and reviews margin guardrails.
6. User reviews vendor stack recommendations generated by the rules engine.
7. User selects or customizes templates for contracts, onboarding, and SOPs.
8. User configures security baseline controls.
9. User prepares launch campaigns and operating assets.
10. User starts tracking baseline KPIs after launch.

## Founder Profile Flow
1. Capture operator background, region, and service motion.
2. Save profile as a reusable recommendation input.
3. Feed recommendation context into pricing and stack policies.

## Service Packaging Flow
1. Define service building blocks.
2. Bundle services into a package.
3. Attach pricing models to the package.
4. Use the package as an anchor for templates, onboarding, and stack guidance.

## Stack Recommendation Flow
1. Load founder profile, business model, package mix, and vendor catalog.
2. Run centralized recommendation policies.
3. Return ranked recommendations with rationale and confidence.
4. Persist accepted recommendations for future workflows.

## Templates Flow
1. Browse template assets by type and version.
2. Select a template based on business model and package posture.
3. Render with variables in a future implementation phase.

## KPI Tracking Flow
1. Choose KPI definitions relevant to the operator stage.
2. Record values by time period.
3. Use metrics in future dashboards and operating reviews.
